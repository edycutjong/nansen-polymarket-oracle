## ūüí° The Problem

Prediction markets show crowd consensus. But the crowd is often wrong.

**Smart Money** ‚ÄĒ hedge funds, whale traders, and institutional wallets ‚ÄĒ frequently takes positions that diverge from the market. These hidden signals are buried in on-chain data that most traders never see.

## ‚ö° The Solution

The Oracle surfaces one actionable insight: **"Where is Smart Money silently betting against the crowd?"**

```
Market Odds say:     12% YES on "Will Bitcoin hit $200K by June 2026?"
Smart Money says:    67% positioned on YES ($6.0M deployed)
Divergence:          ūüĒ• +55 pts ‚ÄĒ SM is massively bullish vs. crowd
```

When Smart Money conviction diverges significantly from market odds, it signals a potential mispricing.

## ūüéĮ How It Works

### Core Pipeline: `scan` ‚Üí `enrich` ‚Üí `analyze`

```
‚ĒĆ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚Ēź     ‚ĒĆ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚Ēź     ‚ĒĆ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚Ēź

‚Ēā markets via     ‚Ēā     ‚Ēā against Nansen  ‚Ēā     ‚Ēā Score           ‚Ēā
‚Ēā Nansen CLI      ‚Ēā     ‚Ēā profiler labels ‚Ēā     ‚Ēā (-100 to +100)  ‚Ēā
‚ĒĒ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚Ēė     ‚ĒĒ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚Ēė     ‚ĒĒ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚Ēė
```

1. **Scan** ‚ÄĒ Fetches active prediction markets from Nansen CLI `research prediction-market` endpoints
2. **Enrich** ‚ÄĒ Cross-references holder addresses against Nansen's `profiler labels` to identify Smart Money wallets (Funds, Smart Traders, 90D Traders, etc.)
3. **Analyze** ‚ÄĒ Calculates a capital-weighted **SM Divergence Score** (-100 to +100):
   - **Positive** = SM is more bullish than market odds
   - **Negative** = SM is more bearish than market odds
   - Markets with |score| ‚Č• 40 are flagged as **ūüĒ• EXTREME** alpha opportunities

### The SM Divergence Score

```typescript
// SM Divergence Score: -100 to +100
// Positive = SM more bullish than market
// Negative = SM more bearish than market

function calculateDivergence(smHolders, allHolders, marketOdds):
  1. Capital-weight each SM holder's position (YES = +weight, NO = -weight)
  2. Normalize weighted conviction from [-1, 1] to [0, 1]
  3. Divergence = (SM_conviction - market_odds) √ó 100
```

| Score | Level | Signal |
|-------|-------|--------|
| ¬Ī40+ | ūüĒ• EXTREME | SM strongly disagrees with market |
| ¬Ī25+ | ‚ö†ÔłŹ HIGH | Notable divergence worth monitoring |
| ¬Ī10+ | ūüďä MODERATE | Mild divergence |
| <¬Ī10 | ‚úÖ LOW/ūü§Ě ALIGNED | SM agrees with market consensus |
